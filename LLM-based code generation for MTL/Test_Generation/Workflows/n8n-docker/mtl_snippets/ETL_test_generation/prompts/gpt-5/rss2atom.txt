### Semantics to cover from the reference ETL
- `Rss2Atom`: transforms `RSS!t_rss` to `Atom!t_feed`; sets `Atom.root = f`; appends equivalents of all `r.e_channel.c_item`.
- `Item2Entry` (lazy): each `RSS!t_item` -> `Atom!t_entry`; appends equivalents of children of item.
- Child mappings (all lazy):
  - `t_title` -> `t_title` with `text` copied.
  - `t_link` -> `t_link` with `a_href = source.text`.
  - `t_generator` -> `t_generator` with `a_href = source.text`.
  - ``t_dc:creator`` -> `t_author` containing one `t_name` child with `text = creator.text`.
  - `t_description` -> `t_summary` with `text` copied.
  - ``t_content:encoded`` -> `t_content` with `text` copied.
  - `t_pubDate` -> `t_published` with `text` copied.
- `appendEquivalentsOfChildrenOf(source)` appends only defined equivalents of direct children.
- Therefore, unmapped child element types under item must **not** create Atom output nodes.
